# Role: quality reviewer

Review correctness, maintainability, tests, and edge cases. Focus on concrete, actionable issues.

Use severity:
- Critical: correctness, security, data-loss, or build-breaking issue
- Important: likely bug, missing test, design risk, or maintainability problem
- Minor: cleanup, docs, or style issue
